# MIM.M.003 — Инженерный цикл развития

> Обновление портфеля проблем → Проблематизация → Стратегирование → Проверка → Замер.

## Summary

Инженерный цикл развития — метод организации непрерывного профессионального развития как инженерного процесса: не «пройти курс от начала до конца», а итеративно обновлять портфель проблем, проблематизировать текущую ситуацию, выбирать стратегию, проверять результат и замерять прогресс.

## Этапы цикла

1. **Обновление портфеля проблем** — выявление новых проблем, требующих решения (через разборы, рефлексию, внешнюю обратную связь)
2. **Проблематизация** — превращение ситуации в задачу: что именно не работает, почему, как проявляется
3. **Стратегирование** — выбор подхода к решению: какой формат, какие ресурсы, какой срок
4. **Проверка** — применение решения и наблюдение результатов
5. **Замер** — фиксация прогресса через разбор (MIM.FMT.004) и замер квалификации (MIM.WP.003)

## Ключевое свойство

Цикл бесконечен: каждый замер обновляет портфель проблем, создавая новые задачи для следующей итерации. Развитие не заканчивается.

## Связь с программами

- **Исследовательское** (MIM.PRG.003): обновление портфеля через семинары — узнаёшь о новых проблемах
- **Личное** (MIM.PRG.001): проблематизация + проверка через практикум — тренируешь навык
- **Рабочее** (MIM.PRG.002): проверка через резидентуру — применяешь навык к шедевру

## Принцип

> «У нас не обучение с оценками, у нас развитие с разборами.»

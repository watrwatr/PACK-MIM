# MIM.M.004 — Квалификационная рамка: три шкалы

> МИМ отслеживает три независимых индикатора квалификации инженера-менеджера. Агентность — первостепенный.

## Назначение

Метод задаёт три шкалы, по которым производится замер квалификации (MIM.WP.003) при каждом разборе (MIM.FMT.004). Шкалы независимы: можно быть методологически дисциплинированным, но иметь слабую агентность.

---

## Шкала 1: Агентность

**Вопрос:** Насколько человек сам инициирует изменения в мире?

| Уровень | Описание |
|---------|----------|
| Минимум | Выполняет одну операцию по поручению и под строгим присмотром |
| Средний | Берёт задачи по своей инициативе, предлагает решения |
| Высокий | Сам ставит проблемы, сам находит решения, присматривает за другими |
| Максимум | Создаёт других агентов, способных быть самостоятельными |

**Типичный режим ошибки:** «Установка на аналитику» — человек способен часами обсуждать содержание руководств, гордится фундаментальными знаниями, но не делает интервенций, не меняет ни себя, ни окружающий мир. Аналитические отчёты ≠ агентность (MIM.D.011).

> Агентность — шкала первостепенного внимания в МИМ.

---

## Шкала 2: Масштаб («радиус влияния», «калибр личности»)

**Вопрос:** Для какой по размеру системы человек показал инициированное им изменение поведения?

| Уровень | Система |
|---------|---------|
| 1 | Я |
| 2 | Команда |
| 3 | Организация |
| 4 | Сообщество |
| 5 | Цивилизация |

**Временной масштаб:** убедить на пять минут ≠ добиться изменений, удержавшихся годы. Оба измерения важны.

**Метрика:** наибольшая граница системы, для которой человек показал **инициированное им** изменение поведения.

---

## Шкала 3: Методологическая дисциплина

**Вопрос:** Насколько выбор методов соответствует ситуации, а не привычкам?

| Уровень | Описание |
|---------|----------|
| Ad hoc | Использует «любимый» метод, не объясняя выбор |
| Compliance | Следует руководствам МИМ как стандартам деятельности |
| SoTA | Выбирает корректный метод по ситуации, объясняет почему и где границы применимости |
| Реформатор | Предлагает улучшения в руководства там, где они плохо срабатывают |

**Ключевой принцип:** Руководства МИМ — стандарты деятельности и мышления (хотя изложены не в формате стандарта). Compliance — начало шкалы, не предел. Руководства — не догма.

**Что проверяется:** использует материалы руководств как описание шагов работы (а не как цитаты), умеет объяснить границы применимости метода, проверяет невязку между предсказаниями теории и наблюдениями.

# Каталог различений: MIM

> Концептуальные границы, нарушение которых создаёт неустранимую путаницу в организации развития.

---

## MIM.D.001 — Семинар ≠ Практикум

| | Семинар | Практикум |
|---|---------|-----------|
| **Главный эффект** | Знакомство с новым знанием | Набивание руки в навыке |
| **Участник** | Слушает, задаёт вопросы | Делает упражнения, получает обратную связь |
| **Результат** | Коллективное понимание | Автоматизация базовых операций |
| **Толерантность к ошибкам** | Вопросы на проблематизацию | Безопасная среда для тренировки |
| **Метафора** | Пришёл на концерт | Играет гаммы |

**Тест:** Если убрать докладчика, мероприятие невозможно → семинар. Если убрать упражнения, мероприятие бессмысленно → практикум.

**Типичная путаница:** «Послушал семинар — теперь умею» (MIM.FM.001). Семинар знакомит, но не тренирует.

**Почему критично:** Ожидание навыка от семинара → разочарование. Ожидание нового знания от практикума → неэффективное использование времени.

## MIM.D.002 — Практикум ≠ Резидентура

| | Практикум | Резидентура |
|---|-----------|-------------|
| **Цель** | Освоение мастерства с нуля | Повышение квалификации в выдаче результата |
| **Результат** | Учебный (внутренний) | Рабочий («шедевр», внешний) |
| **Кто приходит** | Начинающий | Уже профи |
| **Что в зачёт** | «Смотри, как я изменился» | Рабочие успехи |
| **Обязательства** | Перед собой | Перед проектом |

**Тест:** Результат выбрасывается после упражнения → практикум. Результат предъявляется заказчику/коллегам → резидентура.

**Типичная путаница:** Резидент приходит без базовых навыков и пытается сразу делать «шедевр» → провал.

**Почему критично:** Перепутать → отправить новичка в резидентуру (нет базы) или профи в практикум (скука, нет вызова).

---

## MIM.D.003 — Резидентура ≠ Стажировка

| | Резидентура | Стажировка |
|---|-------------|------------|
| **Проект** | Свой (сам выбираешь шедевр) | Выданный (начальник задаёт) |
| **Инициатива** | У резидента | У работодателя |
| **Наставничество** | Review результатов | Обучение на рабочем месте |
| **Результат** | Шедевр (твой выбор) | Выполненные задания |

**Тест:** Кто выбирает, что делать? Ты сам → резидентура. Начальник → стажировка.

**Почему критично:** Путаница → резидент ждёт «задания», вместо того чтобы определить свой шедевр. Или организатор навязывает проект.

## MIM.D.004 — Разбор ≠ Экзамен

| | Разбор (review) | Экзамен |
|---|-----------------|---------|
| **Когда** | Непрерывно (1-2 раза в неделю) | В конце (один раз) |
| **Цель** | Обратная связь + замер | Допуск / оценка |
| **Режим** | Развитие и оценка одновременно | Сначала учёба, потом оценка |
| **Аналог** | Code review, архитектурный комитет | Финальный тест |
| **Ошибки** | Выявляются и исправляются по ходу | Накапливаются до проверки |

**Тест:** Квалификация замеряется при каждой встрече → разбор. Квалификация проверяется в конце → экзамен.

**Типичная путаница:** «У нас же развитие — зачем оценки?» vs «Сейчас учимся, потом экзамен.» В МИМ нет этого разделения: каждый разбор — и развитие, и замер.

**Почему критично:** Разделение режимов «учусь» / «меня оценивают» → участники приносят на разбор «причёсанный» результат вместо сырого. Это убивает обратную связь.

## MIM.D.005 — Формат ≠ Программа

| | Формат | Программа |
|---|--------|-----------|
| **Что это** | Тип мероприятия (как устроено) | Направленность развития (что развиваем) |
| **Примеры** | Семинар, практикум, резидентура, разбор | Личное, рабочее, исследовательское |
| **Различает по** | Главному эффекту | Объекту развития |
| **Связь** | Один формат может быть в разных программах | Одна программа использует разные форматы |

**Тест:** «Какого типа это мероприятие?» → формат. «Что мы развиваем?» → программа.

**Типичная путаница:** «Резидентура — это программа рабочего развития.» Нет: резидентура — формат, а программа рабочего развития использует этот формат как основной, но не единственный.

**Почему критично:** Отождествление → потеря гибкости. Разбор — не программа, а формат, который пронизывает все программы.

## MIM.D.006 — Замещение функций ≠ Уничтожение рабочих мест

| | Замещение функций | Уничтожение рабочих мест |
|---|-------------------|--------------------------|
| **Что происходит** | ИИ берёт конкретные операции (поддержка, рутина) | Массовая безработица, люди не нужны |
| **Что создаётся** | Новые роли, невообразимые заранее | Ничего (пессимистичный нарратив) |
| **Исторический паттерн** | >50% рабочих мест с 1940 — в профессиях, не существовавших в 1940 | Каждая технореволюция порождала панику, но не безработицу |
| **Вывод для развития** | Учись создавать новые роли | Учись «защищаться» от ИИ |
| **Эмпирика** | Customer support: 8.3% → 2.9% новых наймов за 2 года ([a16z, 2025](https://www.a16z.news/p/charts-of-the-week-customer-service)) | — |

**Тест:** «ИИ заменит профессию X» → замещение функции (конкретная операция уходит). «ИИ оставит всех без работы» → ложное обобщение (не подтверждается историей).

**Типичная путаница:** Родитель видит новости «ИИ заменит бухгалтеров» → паника → «надо быстрее в вуз за дипломом». На самом деле: исчезают функции, а не потребность в людях. Тот, кто умеет учиться и создавать новое, создаст роль, которой ещё нет.

**Почему критично:** Эта путаница определяет образовательную стратегию. «Защита от ИИ» → углубление в узкую специализацию (вуз). «Создание новых ролей» → мастерство ученика + проектная работа + экзокортекс (МИМ).

## MIM.D.007 — Принцип ≠ Inductive Bias

| | Принцип | Inductive Bias |
|---|---------|---------------|
| **Система** | S2 (рассудочное мышление) | S1 (интуиция) |
| **Режим** | Осознаваемый, проверяемый | Автоматический, встроенный в архитектуру |
| **Аналог в ML** | Правило в prompt / chain-of-thought | Ограничение архитектуры сети |
| **Скорость** | Медленный (проверки, рассуждения) | Быстрый (мгновенный) |
| **Как формируется** | Изучение, объяснение | Deliberate practice, повторение с обратной связью |

**Тест:** Проверяешь правило сознательно перед применением → принцип (S2). Применяешь автоматически, не задумываясь → inductive bias (S1).

**Типичная путаница:** «Знаю принцип → владею принципом.» Это MIM.FM.001 на более глубоком уровне. Знание принципа (S2) ≠ встроенный bias (S1). Переход S2→S1 требует deliberate practice (MIM.M.001).

**Почему критично:** Объясняет, зачем нужны практикумы и резидентуры, а не только семинары. Семинар знакомит с принципом (S2). Практикум превращает принцип в inductive bias (S1). Без этого перехода принцип остаётся «кабинетным знанием». Источник аналогии: статья «From Kepler to Newton» — простые архитектурные ограничения (biases) определяют, станет ли агент подгонщиком кривых или учёным (Левенчук, 2026-02-18).

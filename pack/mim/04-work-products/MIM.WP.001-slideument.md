# MIM.WP.001 — Слайдомент

> Слайды-как-документ семинара: конспект доклада + материал для AI-агента.

## Summary

Слайдомент — рабочий продукт семинара (MIM.FMT.001), объединяющий свойства слайдов и документа. Используется как конспект для участников и как материал для обсуждения с AI-агентом. Контринтуитивный продукт: критикуется в руководствах по презентациям, но в МИМ это сознательный выбор.

## Характеристики

- Формат: слайды, насыщенные текстом (≠ минималистичные слайды)
- Самодостаточны: читаемы без докладчика
- Длительность доклада: от пары часов до целого дня (6 часов)
- Со временем материал попадает в руководства

## Функции

1. **Конспект** — участник пересматривает материал после семинара
2. **Материал для AI-агента** — можно сразу загрузить и обсудить с AI новые методы
3. **Прото-руководство** — через время содержание формализуется в руководства

## Почему «контринтуитивный»

Обычные руководства по презентациям рекомендуют минимум текста на слайдах. Слайдомент нарушает это правило сознательно: в МИМ важнее, чтобы материал был полным и пригодным для самостоятельного изучения и для AI-агента, чем чтобы слайды были «красивыми».

## Инструментарий

> **Implementation Note.** Конкретный инструмент (Marp, VS Code) — текущая реализация. Доменное требование: слайдомент должен быть plain-text (версионируемый, редактируемый ИИ-агентом). Детали: [C2.IT-Platform](../../../DS-ecosystem-development/C.IT-Platform/C2.IT-Platform/).

Основной инструмент: **Marp** — Markdown → слайды (PDF, HTML, PPTX).

| Компонент | Назначение | Установка |
|-----------|-----------|-----------|
| **Marp CLI** | Экспорт: `marp --pdf slides.md` | `npm install -g @marp-team/marp-cli` |
| **Marp for VS Code** | Живой предпросмотр слайдов | VS Code → Extensions → «Marp for VS Code» |

**Workflow:** Markdown-файл с frontmatter `marp: true` и разделителем `---` между слайдами → предпросмотр → экспорт в PDF. Стилизация через CSS. Совместим с Git: версии, диффы, правки.

**Почему plain text:** Слайдомент — текстовый РП. Бинарные форматы (PowerPoint/Google Slides) не версионируются в Git и не редактируются ИИ-агентом. Plain text = полная интеграция с IWE.

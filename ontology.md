# Ontology: Мастерская инженеров-менеджеров (MIM)

> Domain ontology per SPF.SPEC.002.
> Реестр видов сущностей, ключевых терминов и связей для Pack MIM.
> Каждое понятие связано с родительским из SPF base ontology (SPF.SPEC.002, Section 2).

---

## 1. Entity Types

> Все виды сущностей Pack: базовые (из SPF) + расширенные (определены этим Pack).

| Code | Type | FPF/SPF Concept | Definition | ≠ (what it is NOT) | Source |
|------|------|-----------------|------------|---------------------|--------|
| `M` | Method | U.Method | Способ организации развития, производящий определённый эффект | ≠ формат, ≠ инструмент | SPF (base) |
| `WP` | Work Product | U.Work + U.Episteme | Наблюдаемый результат формата или метода | ≠ описание метода | SPF (base) |
| `FM` | Failure Mode | — (SPF-specific) | Типичное нарушение в организации развития | ≠ ошибка участника | SPF (base) |
| `D` | Distinction | A.7 Strict Distinction | Граница между форматами, программами или подходами | ≠ факт, ≠ определение | SPF (base) |
| `R` | Role | U.RoleAssignment | Функциональная позиция в формате мероприятия | ≠ должность в МИМ | SPF (base) |
| `FMT` | Format | U.Method (specialized) | Организационный шаблон мероприятия развития, различаемый по главному эффекту | ≠ программа, ≠ метод | Pack (extended) |
| `PRG` | Program | U.System (specialized) | Тип программы развития, различаемый по направленности развиваемого объекта | ≠ формат, ≠ курс | Pack (extended) |

## 2. Domain Glossary

| Формат мероприятия | Организационный шаблон, различаемый по главному эффекту: знание, навык, результат, обратная связь | U.Method | MIM.FMT.* |

| Практикум | Формат тренировки навыка через deliberate practice | U.Method | MIM.FMT.002 |
| Резидентура | Формат получения рабочего результата (шедевра) с наставничеством | U.Method | MIM.FMT.003 |

| Принцип (S2) | Осознаваемое, проверяемое ограничение на мышление. Действует медленно, через рассуждения | A.7 Strict Distinction | MIM.D.007 |
| Inductive bias (S1) | Встроенное в архитектуру ограничение, действующее автоматически и быстро. Формируется из принципов через practice | — (ML concept) | MIM.D.007 |
| Программа усиления интеллекта | Мета-цель МИМ: не широкий набор курсов, а фокусированная программа, усиливающая интеллект через трансдисциплины и deliberate practice | U.System | MIM.PRG.* |

## 3. Relationships Between Types

| Subject | Relationship | Object | Example |
|---------|-------------|--------|---------|
| Format (FMT) | produces → | Work Product (WP) | MIM.FMT.001 → MIM.WP.001 (семинар → слайдомент) |
| Format (FMT) | is primary in → | Program (PRG) | MIM.FMT.002 → MIM.PRG.001 (практикум → личное развитие) |
| Method (M) | applied in → | Format (FMT) | MIM.M.001 → MIM.FMT.002 (deliberate practice → практикум) |
| Role (R) | participates in → | Format (FMT) | MIM.R.001 → MIM.FMT.004 (наставник → разбор) |
| Failure Mode (FM) | violates ← | Distinction (D) | MIM.FM.001 ← MIM.D.001 (послушал=научился ← семинар≠практикум) |
| Distinction (D) | deepens → | Failure Mode (FM) | MIM.D.007 → MIM.FM.001 (принцип≠bias объясняет послушал=научился) |
| Method (M) | converts via → | Distinction (D) | MIM.M.001 → MIM.D.007 (deliberate practice превращает принцип S2 в bias S1) |
| Format (FMT) | contains → | Format (FMT) | MIM.FMT.004 inside MIM.FMT.002 (разбор внутри практикума) |
| Program (PRG) | uses → | Format (FMT) | MIM.PRG.002 uses MIM.FMT.003 + MIM.FMT.004 |
| Format (FMT) | composes → | Format (FMT) | MIM.FMT.005 composes MIM.FMT.001 + MIM.FMT.002 + MIM.FMT.004 |
| Role (R) | qualifies into → | Role (R) | MIM.R.003 → MIM.R.004 (резидент → мастер, через шедевр) |
| Role (R) | can become → | Role (R) | MIM.R.004 → MIM.R.001 (мастер → наставник) |

## 4. Type Hierarchy

```
МИМ (организация)
├── Программы развития (PRG)
│   ├── Личное развитие (PRG.001)
│   ├── Рабочее развитие (PRG.002)
│   └── Исследовательское развитие (PRG.003)
│
├── Форматы мероприятий (FMT)
│   ├── Семинар (FMT.001) ─── основной в PRG.003
│   ├── Практикум (FMT.002) ── основной в PRG.001
│   ├── Резидентура (FMT.003) ── основной в PRG.002
│   ├── Разбор (FMT.004) ───── пронизывает все
│   └── Марафон (FMT.005) ─── композитный (семинар+практикум+разбор)
│
├── Методы (M)
│   ├── Deliberate Practice (M.001) ── в практикуме
│   ├── Наставничество через разбор (M.002) ── в резидентуре и практикуме
│   └── Инженерный цикл развития (M.003) ── сквозной
│
├── Рабочие продукты (WP)
│   ├── Слайдомент (WP.001) ── продукт семинара
│   ├── Шедевр (WP.002) ── продукт резидентуры
│   └── Замер квалификации (WP.003) ── продукт разбора
│
└── Роли (R)
    ├── Наставник (R.001)
    ├── Докладчик (R.002)
    ├── Резидент (R.003)
    └── Мастер (R.004) ─── прошёл резидентуру, может стать наставником
```

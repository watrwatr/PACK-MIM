# Инструкции для PACK-MIM

> Общие инструкции: `~/Github/CLAUDE.md`

## Домен

Pack «Мастерская инженеров-менеджеров» (MIM) — организация и методики бесконечного профессионального развития.

Два слоя:
1. **SPF** — фреймворк (структура Pack, виды сущностей, кодирование)
2. **MIM** — домен (форматы мероприятий, программы развития, принципы, роли)

## Жёсткие запреты

- **Не путать формат и программу** — формат (семинар, практикум) ≠ программа (личное, рабочее развитие)
- **Не писать дидактику** — Pack описывает ЧТО существует, а не КАК учить
- **Не путать МИМ и сообщество** — МИМ = организация (сотрудники, бюджет); сообщество = объединение единомышленников
- **Не путать разбор и экзамен** — разбор = замер + обратная связь; экзамен = оценка + допуск
- **Не копировать downstream** — конкретные расписания, потоки, мотивация → downstream

## Кодирование сущностей

Контекст: `MIM` | Формат: `MIM.<ВИД>.<НОМЕР>`

### Расширенные виды (этот Pack)

| Код | Вид | Описание |
|-----|-----|----------|
| `FMT` | Format | Формат мероприятия развития |
| `PRG` | Program | Тип программы развития |

### Базовые виды (из SPF)

M (Method), WP (Work Product), FM (Failure Mode), D (Distinction), R (Role), CHR (Characteristic)

## Имена файлов

```
MIM.FMT.001-seminar.md
MIM.D.001-seminar-vs-practicum.md
MIM.M.001-deliberate-practice.md
```

## Процедуры

### Добавление сущности

1. Проверить: нет ли уже в Pack
2. Определить вид (FMT/PRG/M/WP/FM/D/R)
3. Назначить код (следующий номер)
4. Создать файл по шаблону
5. Обновить 00-pack-manifest.md (Entity Index)
6. Обновить ontology.md (если новый термин)

### Pre-commit чеклист

- [ ] Код уникален (MIM.ВИД.NNN)
- [ ] Файл в правильной папке
- [ ] Frontmatter заполнен (id, name, status, related)
- [ ] Summary ≤ 2 предложения
- [ ] Нет дидактики
